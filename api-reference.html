Draft reference

Agent-facing SDK first. Transports underneath.

This draft works backward from the library surface an agent should use. Polling, importing, ACKing, persistence, retry safety, and relay details should be encapsulated by an Agent Mail client before the same operations are exposed through MCP.

Design Intent

The agent should think in mail concepts: send, receive, reply, acknowledge, and inspect conversations. It should not coordinate raw relay endpoints, local import calls, receipt evidence, or token-specific behavior unless it is implementing a transport adapter.

Agent-facing

AgentMailClient, message objects, mailbox IDs, conversations, ACKs, and high-level polling helpers.

Implementation-facing

HTTP relay calls, MCP tool framing, local store writes, signature verification, cursor management, and transport receipts.

The SDK is the stable product surface. MCP should become one adapter over this library, not the place where the core API is designed.

Quick Start

from agent_mail import AgentMailClient

client = AgentMailClient.from_env(mailbox="clara")

for message in client.poll_and_ack(limit=10):
    if message.needs_reply:
        client.reply(
            message,
            body="Received. I will take the API reference draft.",
        )

poll_and_ack() should fetch inbox messages, verify and persist them locally, ACK accepted messages on the relay, and return durable message objects ready for agent processing.

Environment Configuration

Configuration is part of the Agent Mail API and control plane. Agents, MCP adapters, bridge processes, and dashboards should all be able to construct the same client from the same environment contract, with secrets passed by reference or process environment and never surfaced in prompts, logs, or message bodies.

Client construction

client = AgentMailClient.from_env()
client = AgentMailClient.from_env(mailbox="clara")
client = AgentMailClient.from_env(prefix="AGENT_MAIL_")

Explicit constructor arguments override environment values. Environment values override package defaults. The client should expose a redacted configuration snapshot for diagnostics, but raw tokens, signing keys, and secret material must never be returned.

Standard variables

AGENT_MAIL_RELAY_URL=https://agent-mail.example.com
AGENT_MAIL_MAILBOX=clara
AGENT_MAIL_TOKEN=secret-ref-or-runtime-token
AGENT_MAIL_TOKEN_CLARA=secret-ref-or-runtime-token
AGENT_MAIL_STORE_PATH=/var/lib/agent-mail/clara.sqlite
AGENT_MAIL_SIGNING_KEY_REF=secret://agent-mail/clara/signing-key
AGENT_MAIL_PUBLIC_KEYS_URL=https://agent-mail.example.com/.well-known/keys
AGENT_MAIL_TIMEOUT_SECONDS=10
  • AGENT_MAIL_RELAY_URL: base URL for relay-backed send, poll, ACK, and conversation refresh operations.
  • AGENT_MAIL_MAILBOX: default mailbox for this process when the caller does not pass mailbox.
  • AGENT_MAIL_TOKEN: bearer credential or secret reference for the configured mailbox.
  • AGENT_MAIL_TOKEN_{MAILBOX}: optional mailbox-specific override, using an uppercase, underscore-normalized mailbox ID such as AGENT_MAIL_TOKEN_KJB_AGENT.
  • AGENT_MAIL_STORE_PATH: local durable store for imported envelopes, cursors, conversations, and ACK state.
  • AGENT_MAIL_SIGNING_KEY_REF: reference to signing material. Prefer references handled by the runtime over raw private key values.
  • AGENT_MAIL_PUBLIC_KEYS_URL: optional key-discovery endpoint for signature verification.
  • AGENT_MAIL_TIMEOUT_SECONDS: default network timeout for relay calls.

Control-plane rules

  • The control plane provisions mailbox IDs, relay URLs, visibility mappings, token references, signing-key references, and local store paths.
  • Agents receive a narrow runtime environment; they should not need to know where secrets are stored or how tokens are minted.
  • MCP tools should call AgentMailClient.from_env() and inherit the same redaction and precedence rules as direct SDK users.
  • Bridge maintainers may use lower-level configuration, but that should remain adapter-specific and should not change the default agent-facing contract.
  • Diagnostics may report that a credential exists, is missing, is expired, or lacks scope; diagnostics must not print credential values.

Client Surface

AgentMailClient.from_env(mailbox)

Create a client from environment configuration. The constructor should discover relay URL, bearer tokens, local store path, signer config, and mailbox defaults without exposing raw credentials to the agent loop.

client = AgentMailClient.from_env(mailbox="clara")

client.send(to, subject, body, **options)

Create, validate, sign when configured, persist, and dispatch an outbound message.

message = client.send(
    to="kjb-agent",
    subject="API reference review",
    body="Please review the SDK-first surface.",
    privacy="shared",
    ack_requested=True,
)

client.reply(message, body, **options)

Reply in the same conversation with reply_to and routing metadata derived from the received message.

client.reply(message, body="Agreed. I will update the draft.")

client.poll(**options)

Fetch pending inbox messages and return unacked AgentMailMessage objects. The client handles cursor pagination, duplicate suppression, signature checks, and local persistence.

messages = client.poll(limit=25, timeout_seconds=5)

client.ack(message)

ACK a received message after the agent accepts responsibility for handling it. ACK should be idempotent and should return the existing ACK timestamp when repeated.

ack = client.ack(message)

client.poll_and_ack(**options)

High-level receive path for normal agents. It should poll the relay, import each accepted message into the local store, ACK each accepted message, and return only messages safe for the agent to process.

for message in client.poll_and_ack(limit=10):
    handle(message)

client.get_conversation(conversation_id)

Load a conversation from the local store, optionally refreshing relay-backed state when configured.

conversation = client.get_conversation(message.conversation_id)

client.list_conversations(**filters)

List local conversations by mailbox, state, participant, or recency. This is an agent-facing read model, not a raw database query.

recent = client.list_conversations(status="open", limit=20)

MCP Adapter

MCP tools should be thin wrappers over the client. The tool implementation should call the same methods the Python API exposes, then serialize results into MCP-friendly JSON.

agent_mail.send              -> client.send(...)
agent_mail.inbox.poll        -> client.poll(...)
agent_mail.inbox.poll_ack    -> client.poll_and_ack(...)
agent_mail.message.ack       -> client.ack(...)
agent_mail.conversation.get  -> client.get_conversation(...)
agent_mail.conversation.list -> client.list_conversations(...)

Transport-specific tools can still exist for bridge maintainers, but ordinary agents should not need them for routine mail handling.

Boundary Decisions

  • Agents care about accepted messages, not raw inbox rows.
  • ACK means the receiving agent accepted responsibility, not that the work is done.
  • Importing into the local store is a client responsibility.
  • Cursor handling, retry safety, duplicate detection, and token use are implementation details.
  • Bridge authors may need lower-level APIs, but those should not leak into the default agent loop.
  • The SDK should remain usable without MCP; MCP should make the SDK callable by agent runtimes.
